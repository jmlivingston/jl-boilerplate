REST API Definitions

Note: Depending on the data source, the name of the ID (ID, id, _id, etc.) may be different. Client side code needs to be aware of the datasource and handle accordingly. We could do data tranformations, but in many cases this can cause performance problems.

ROUTE NAMES

  • issues-firebase
  • issues-json
  • issues-mongo
  • issues-sql

ROUTES

GET

Path Example Return Values
/api/issues-firebase
/api/issues-json
/api/issues-mongo
/api/issues-sql 
{ items: [ { _id: 1, FirstName: 'Bill' }, { _id: 2, FirstName: 'Bob' } ] }
/api/issues-firebase/:id
/api/issues-json/:id
/api/issues-mongo/:id
/api/issues-sql/:id 
{ item: { _id: 1, FirstName: 'Bill' } }

POST

Path Example Return Values
/api/issues-firebase
/api/issues-json
/api/issues-mongo
/api/issues-sql 
{ id: 1 }

//or

{ ids: [ 1, 2 ] }

PUT

Path Example Return Values
/api/issues-firebase/:id
/api/issues-json/:id
/api/issues-mongo/:id
/api/issues-sql/:id 
{ id: 1, success: true }

DELETE

Path Example Return Values
/api/routename/:id 
{ id: 1, success: true }

ERRORS

If there is any error, it will be returned in an error property. On the front end, always check this first. 

{ error: 'there was an error' }
